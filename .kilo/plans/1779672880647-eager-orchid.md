# Plan: "Show Best Candidate" Feature for Matching Page

## Overview
Add a button that finds and displays the best matching candidate from selected candidates.

## Implementation Steps

### 1. Add State for Best Candidate Modal
- Add `showBestModal` state (boolean) to control modal visibility
- Add `bestCandidate` state to store the selected best candidate data

### 2. Create `findBestCandidate` Function
- Filter selected candidates that have scores (`scoredIds`)
- Get matching score for each using `getMatchingScoreForRow`
- Return the candidate with highest score
- Handle edge cases: no selection, no scored candidates

### 3. Add "Show Best Candidate" Button
- Place next to "Run Matching" button
- Disabled when: no JD selected, no candidates selected, or no scored candidates
- Opens modal when clicked

### 4. Create Best Candidate Modal Component

- Title: "Best Match"
- Display: candidate name, position, experience, matching score badge
- Show score breakdown (bar scores)
- Show candidate details similar to expanded view

### 5. Update Matching Page (`app/matching/page.tsx`)
- Import Modal component
- Add click handler for the button
- Add modal JSX at end of component

### Key Logic:
```typescript
const findBestCandidate = () => {
  const selectedCandidates = filtered.filter(c => selectedIds.has(c.id));
  const scoredSelected = selectedCandidates.filter(c => scoredIds.has(c.id));
  
  if (scoredSelected.length === 0 || !selectedJd) return null;
  
  return scoredSelected.reduce((best, current) => {
    const bestScore = getMatchingScoreForRow(best, selectedJd.id, { jd: selectedJd });
    const currentScore = getMatchingScoreForRow(current, selectedJd.id, { jd: selectedJd });
    return currentScore > bestScore ? current : best;
  });
};
```
