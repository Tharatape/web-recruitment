# Plan: Connect Admin Activity Log Table to Filters

## Problem
The Admin Activity page has filter dropdowns (Date Period, Status, Action, Recruiter, Search) but the log table displays static mock data and doesn't actually filter against the database.

## Current State Analysis

### Database Schema (`data/db/schema.sql`)
- `activity_logs` table has: `id`, `candidate_id`, `date`, `time`, `recruiter_id`, `status_id`, `note`
- Missing: `action_type` column needed for "Action" filter

### API (`app/api/activity/route.ts`)
- Currently supports: `days`, `limit`, `offset` query parameters
- Missing: `status`, `action_type`, `recruiter`, `search` filter support
- Already selects `action_type` in query but column doesn't exist in schema

### Frontend (`app/admin-activity/page.tsx`)
- Has filter state variables but they're not connected to API
- Table displays empty `activities` array instead of fetched data

## Implementation Plan

### Step 1: Update Database Schema
- Add `action_type TEXT` column to `activity_logs` table in `schema.sql`
- Action type values should be: "Change Status", "Matching", "Create/Edit JD", "AI Opinion"

### Step 2: Update Seed Logic
- Modify `data/db/seed.ts` to insert `action_type` when seeding activity logs
- Map existing log entries to appropriate action types based on status transitions

### Step 3: Update API Endpoint
- Add filter parameters: `status`, `action_type`, `recruiter`, `search`
- Build dynamic SQL query with WHERE clauses for each filter
- Return filtered activity data with proper joins

### Step 4: Update Frontend
- Add `useEffect` to fetch activities from `/api/activity` on mount and filter change
- Pass filter values as query parameters to the API
- Map API response to display in the table

## Technical Details

### API Query Parameters
```
GET /api/activity?days=7&status=Shortlisted&action_type=Change Status&search=John&limit=50&offset=0
```

### SQL Filter Logic (in API)
```sql
SELECT ... FROM activity_logs al
LEFT JOIN candidates c ON al.candidate_id = c.id
LEFT JOIN owners o ON al.recruiter_id = o.id
LEFT JOIN statuses s ON al.status_id = s.id
-- Add WHERE clauses for each non-empty filter parameter
```

### Filter to Database Mapping
- Date Period → `days` parameter (already exists)
- Status → `status` filter on `s.name` column
- Action → `action_type` filter on `al.action_type` column
- Recruiter → `recruiter` filter on `o.name` column
- Search → `search` filter on `c.name`, `c.email`, or `al.note`

## Files to Modify
