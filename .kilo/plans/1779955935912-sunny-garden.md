## 3. Database Schema & Data Modeling

### Enhanced Candidates Table Schema (SQLite)

```sql
-- Reference tables for normalization
CREATE TABLE IF NOT EXISTS departments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL
);

CREATE TABLE IF NOT EXISTS types (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL CHECK (name IN ('Staff', 'Contract'))
);

CREATE TABLE IF NOT EXISTS application_statuses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL
  -- Values: Applied, In Review, Shortlisted, etc.
);

CREATE TABLE IF NOT EXISTS interview_statuses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL
  -- Values: Scheduled, Completed, Passed, Failed
);

CREATE TABLE IF NOT EXISTS offer_statuses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL
  -- Values: Pending, Extended, Accepted, Declined
);

-- Enhanced Candidates table
CREATE TABLE IF NOT EXISTS candidates (
  id TEXT PRIMARY KEY,
  unique_id TEXT UNIQUE NOT NULL,
  date_applied TEXT NOT NULL,
  position TEXT NOT NULL,
  type_id INTEGER NOT NULL,
  department_id INTEGER,
  experience INTEGER NOT NULL,
  degree TEXT,
  major TEXT,
  toeic_score INTEGER,
  age INTEGER,
  bmi REAL,
  weight INTEGER,
  height INTEGER,
  application_status_id INTEGER,
  interview_status_id INTEGER,
  offer_status_id INTEGER,
  hired INTEGER DEFAULT 0 CHECK (hired IN (0, 1)),
  owner_id INTEGER,
  FOREIGN KEY (type_id) REFERENCES types(id),
  FOREIGN KEY (department_id) REFERENCES departments(id),
  FOREIGN KEY (application_status_id) REFERENCES application_statuses(id),
  FOREIGN KEY (interview_status_id) REFERENCES interview_statuses(id),
  FOREIGN KEY (offer_status_id) REFERENCES offer_statuses(id),
  FOREIGN KEY (owner_id) REFERENCES owners(id)
);
```

### TypeScript Interface (data/repositories/kpiRepository.ts)

```typescript
export interface KpiCandidate {
  id: string;
  unique_id: string;        // Format: app-XXXXX (e.g., app-00001)
  date_applied: string;     // ISO date string (YYYY-MM-DD)
  position: string;
  type: 'Staff' | 'Contract';
  department: string | null;
  experience: number;       // Years
  degree: string | null;
  major: string | null;
  toeic_score: number | null;
  age: number | null;
  bmi: number | null;
  weight: number | null;
  height: number | null;
  application_status: string | null;
  interview_status: string | null;
  offer_status: string | null;
  hired: boolean;
  owner: string | null;
}

export interface KpiAggregations {
  positionDistribution: Array<{ name: string; value: number }>;
  educationDistribution: Array<{ name: string; value: number }>;
  experienceDistribution: Array<{ name: string; value: number }>;
  ageDistribution: Array<{ name: string; value: number }>;
  bmiDistribution: Array<{ name: string; value: number }>;
  heightDistribution: Array<{ name: string; value: number }>;
  totalCandidates: number;
  averageExperience: number;
}
```

### Unique ID Generation
- Format: `app-XXXXX` where XXXXX is a zero-padded 5-digit sequence
- Generated via trigger or application logic on INSERT
- Example: `app-00001`, `app-00002`, ..., `app-99999`

### Indexes for KPI Queries
```sql
CREATE INDEX IF NOT EXISTS idx_candidates_date_applied ON candidates(date_applied);
CREATE INDEX IF NOT EXISTS idx_candidates_owner ON candidates(owner_id);
CREATE INDEX IF NOT EXISTS idx_candidates_type ON candidates(type_id);
CREATE INDEX IF NOT EXISTS idx_candidates_department ON candidates(department_id);
CREATE INDEX IF NOT EXISTS idx_candidates_unique_id ON candidates(unique_id);
```

### API Endpoints (app/api/kpi/route.ts)

```

  Response: { aggregations: KpiAggregations }

GET /api/kpi/export
  Query params: search, dateFrom, dateTo, owner
  Response: application/vnd.ms-excel or text/csv
```
